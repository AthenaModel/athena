DESCRIPTION

athenawb ?options...? ?filename.adb?

Athena Workbench is the desktop Athena simulation application. Details of the simulation model are to be found in the Athena Analyst's Guide and related documents, rather than here. The models and algorithms, along with tools for scenario-building, are found in athena(n).

athenawb(1) is a documentic-centric application, like a word-processor; here, however, the documents are Athena scenario files, which conventionally have a ".adb" extension. If a scenario file name is given on the command line, it will be opened automatically.

Athena Workbench is usually invoked from the Windows Start Menu or a desktop shortcut; it may also be invoked from a Windows console command line, in which case the following options may be used:

-scratch dir
Names a directory to use for writing runtime files to. Log files and temporary database files are written here. If left out, Athena defaults to a temporary directory defined by the operating system. For Windows systems, this is typically a directory under the user's home directory.
-script filename.tcl
Names an Athena executive script to be executed at start-up, immediately after loading scenario file (if any). This option is usually used to execute exported order scripts.
-ignoreuser
This option is intended for use during testing; it tells the application to ignore user preference settings, so that the tests always begin with the application in a known state.
-url url
If given, Athena will show the url in the Detail Browser on start-up.

The application has extensive on-line help; invoke the application and see the Help menu.

MODS

It is possible to patch Athena post-delivery by means of Athena mod files. To use a mod, just copy the mod file to the installation's mods/ directory, typically ~/athena/mods.

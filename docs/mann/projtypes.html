DESCRIPTION

This module defines data types used by the Athena code base. Each type is defined as a type ensemble command.

TYPES

boolean validate flag

Boolean flag type. Accepts any of the standard Tcl boolean values.

Validates the flag, returning either 1 for true values and 0 for false values. On error, throws an error with -errorcode INVALID.

This call differs from snit::boolean in that it converts the value to canonical form, and throws INVALID on error.

ecomparatorx subcommand ?args?

An enumx(n) type, representing a variety of comparisons used by Athena conditions:

Index Name longname
0 EQ equal to
1 AE approximately equal to
2 GE greater than or equal to
3 GT greater than
4 LE less than or equal to
5 LT less than

The type has one unique subcommand:

ecomparatorx compare x comp y
Compares numeric value x with numeric value y according to ecomparator value comp, returning 1 if the comparison is true and 0 otherwise.

Index Name Long Name

1 DISPLACED Displaced
2 IN_CAMP In Camp

The type has one unique subcommand:

Moves the item to the place in the list indicated by by where, which is an emoveitem value, and returns the new list.

hexcolor validate color

Hex color type. A hex color specification is a string "#rrggbb" where rr, gg, and bb are two-digit hexadecimal numbers representing the intensity of the RGB color channels.

Validates and returns the color; the hex-digits are upper-cased. On error, throws an error with -errorcode INVALID.

idays validate days

Integer days type: a non-negative integer number.

Validates and returns days. On error, throws an error with -errorcode INVALID.

ident validate name

Identifier type. An identifier is a string consisting only of letters and digits, beginning with a letter.

Validates and returns the name. On error, throws an error with -errorcode INVALID.

iminlines validate quantity

The minimum value for the cli.maxlines preferences parameter. An integer number no less than 100.

Validates and returns quantity. On error, throws an error with -errorcode INVALID.

ipositive validate quantity

Integer quantity type: a positive integer number.

Validates and returns quantity. On error, throws an error with -errorcode INVALID.

iquantity validate quantity

Integer quantity type: a non-negative integer number.

Validates and returns quantity. On error, throws an error with -errorcode INVALID.

money validate dollars
Money type; the dollars value must be a string accepted by the marsmisc(n) moneyscan function. The value is returned as a normal floating point value.
polygon validate coords
Validates the polygon described by the coords, which is a flat list of x and y coordinates. It may be passed as a single argument, or as one argument per coordinate. On error, throws an error with -errorcode INVALID.
qcredit

This quality is used to assess the credit an actor receives for provision of service to civilian groups. qcredit values have the bounds shown in the following table:

Name  Long Name Value Bounds
M  Most  0.75  0.50 < value <= 1.00
S  Some  0.35  0.20 < value <= 0.50
N  Negligible  0.10  0.00 < value <= 0.20
qsecurity

This quality is use to rate a group's security in a neighborhood. Numerically, qsecurity values are formatted as "%4d" and have the bounds shown in the following table:

Name  Long Name Value Bounds
H  High  60  25 < value <= 100
M  Medium  15  5 < value <= 25
L  Low  -10  -25 < value <= 5
N  None  -60  -100 < value <= -25
rgain validate value
A range(n) representing a gain control setting: a non-negative floating-point value. On error, throws an error with -errorcode INVALID.
rpercent validate value
A range(n) representing a non-negative percentage, e.g., a non-negative floating-point value between 0.0 and 100.0, inclusive. On error, throws an error with -errorcode INVALID.
rpercentpm validate value
A range(n) representing a positive or negative percentage, e.g., a floating-point value between -100.0 and 100.0, inclusive. On error, throws an error with -errorcode INVALID.
rposfrac validate value
A range(n) representing a positive fraction from 0.01 to 1.0, inclusive. On error, throws an error with -errorcode INVALID.
rrate validate value
A range(n) representing a rate, e.g., events/day: a non-negative floating-point value. On error, throws an error with -errorcode INVALID.
unitname validate name

Identifier type. A unit name is a string consisting only of letters and digits, "-", or "/", beginning with a letter.

Validates and returns the name. On error, throws an error with -errorcode INVALID.

weight validate value

A weight is a non-negative floating-point value, used as a weighting factor.

Validates the value, returning it on success. On error, throws an error with -errorcode INVALID.

ENVIRONMENT

Requires Tcl 8.5 or later.

To use this package in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.

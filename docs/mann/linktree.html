SYNOPSIS

package require projectgui 6.3.3
namespace import ::projectgui::*
  • linktree pathName ?options...?
  •     -changecmd command
  •     -errorcmd command
  •     -lazy flag
  •     -url url
  • pathName cget option
  • pathName configure option value...
  • pathName get
  • pathName refresh
  • pathName set url

DESCRIPTION

The linktree(n) widget displays a tree of links, each with an icon and label. The top-level set of links is read from a mydomain(i) URL returning a tcl/linkdict given the -url. The widget will follow these links recursively until the entire tree is built.

If -lazy is true, the tree is built lazily; linktree(n) doesn't add child links until a parent link is expanded by the user. For large trees, this can be much faster.

COMMANDS

This module defines the following commands.

linktree pathName ?options...?

Creates a new linktree widget named pathName, and returns pathName. The widget supports the following options, in addition to those provided by the underlying frame widget:

-changecmd command
Specifies a command to be called whenever the selected entity changes, for any reason whatsoever (including explicit calls to set. The newly selected entity's name is appended to the command as an argument.
-errorcmd command
Specifies a command to be called with one argument, an error message, if the widget catches an error while retrieving the -url or one of its subsidiary links.
-lazy flag
If true, the widget will retrieve child links in a lazy fashion, e.g., subtrees will not be populated until the parent item is expanded by the user. Otherwise, the entire tree is populated immediately on refresh. Defaults to false.
-url url
Specifies the URL at which the widget can retrieve a tcl/linkdict. The widget will attempt to recursively retrieve a tcl/linkdict for each link in the returned dictionary.

INSTANCE COMMAND

linktree(n) widgets have the following subcommands:

pathName cget option
Returns the value of the specified option.
pathName configure option value...
Sets the value of one or more of the object's options.
pathName get
Returns the URL of the currently selected entity, or "" if none.
pathName refresh
Causes the widget to reload the displayed entities from the server. Call this when the set of entities to display has changed. Note that this call will not change the open/close state of the entity type folders, nor will it change the selected entity, unless the selected entity no longer exists.
pathName set url

Selects the entity with the given url. If url is unrecognized, or is the empty string, the selection is cleared.

Note that a sequence of calls to set with the same url will result in at most one call of the -changecmd.

ENVIRONMENT

projectgui(n) requires Tcl 8.5 or later.

To use projectgui(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.

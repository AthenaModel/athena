DESCRIPTION

The comparison(n) object compares two scenarios for significant differences, and returns a comparison(n) object that contains them. The differences are captures as a set of vardiff(n) objects, each of which represents a significant difference in a particular variable, e.g., a significant different in the mood of a particular civilian group. Then, the user may produce a "causality" chain for a particular significant output, which traces the difference back through the output variable's inputs, to their inputs and beyond; these differences are also captured as vardiff(n) objects.

The vardiff(n) class is an abstract base class; it is subclassed for each of the important variable types, e.g., group mood, neighborhood mood, and average neighborhood security.

This man page describes the conventions and methods common to all vardiff(n) subclasses.

METADATA

Each vardiff(n) subclass is expected to define the following metadata using the meta class definition statement.

meta type vartype
States the variable type name, e.g., sat, nbmood. By convention, this will also be the bare name of the subclass, e.g., ::athena::vardiff::sat, ::athena::vardiff::nbmood.
meta category category
States the variable's PMESII category, e.g., political, military, or social.
meta normfunc func
Specifies the normalization function to use when scoring a set of variables of this type. The func may be a constant number or one of a number of symbols; see normalizer for the full set and for their meanings.
meta afactors dict
Specifies a dictionary of A factors by input variable type, e.g., the mood type depends on the sat and population types. The A factor is a factor used to rate the type's importance relative to other variable types in the same context as an aid to scoring. The nominal value is 1.0.
meta leaf boolean
If 1, the vardiff is a leaf in the causality tree, i.e., it either has no inputs or input chaining has not yet been implemented. If 0, inputs can and will be chained.

COMMANDS

vardiff(n) provides the following commands:

::athena::vardiff create obj comp keydict val1 val2
::athena::vardiff new comp keydict val1 val2

Creates a new vardiff(n) instance, returning the fully qualified object name. The object exists in the context of comparison(n) object comp. Instances of this type of variable are distinguished by their key dictionary, keydict, which names the relevant keys and their values, e.g., {g SHIA c SFT}. The vardiff is created for values val1 and val2, from the two scenarios being compared.

NOTE: Subclass constructors will take the relevant key values as additional arguments, and construct the keydict internally. Thus, the argument list for a sat vardiff might look like this: {comp 25.5 15.5 SHIA SFT}.

PUBLIC INSTANCE COMMANDS

Each instance of vardiff(n) provides the following subcommands:

obj context
Subclasses should override this method as needed. Returns a string of context information about the difference, for example the numeric range.
obj delta
For variables with numeric values, returns the absolute difference between val1 and val2. For variables with symbolic values, returns some number representing the degree of difference; or, minimally, a constant 1.0.
obj fancy value
Subclasses should override this method as needed. Given a value of the variable's type, formats it for display in a "fancy" way, i.e., includes symbolic values and units as well as the bare number.rounds decimals to one or two decimal places. By default, calls format.
obj fancy1
Returns the result of calling fancy on val1.
obj fancy2
Returns the result of calling fancy on val2.
obj fmt1
Returns the result of calling format on val1.
obj fmt2
Returns the result of calling format on val2.
obj format value
Subclasses should override this method as needed. Given a value of the variable's type, formats it for display, i.e., rounds decimals to one or two decimal places. By default, returns value unchanged.
obj gotInputs
Returns 1 if we've computed significant inputs for this object, and 0 otherwise.
obj huddle
Returns a huddle(n) object corresponding to obj. It has a field for each entry in the view, plus one for the dictionary of of input names and scores.
obj inputs
Returns a dictionary of the input vardiff(n) objects with their scores in the context of this variable, computing them if necessary.
obj key name
Returns the value of the named key.
obj keys
Returns the names of the variable's key parameters in canonical order.
obj keydict
Returns the variable's key dictionary.
obj name
Returns the variable's name, which is a qualified entity ID (qid) based on the variable type and the key values. For example, the satisfaction of group SHIA with concern SFT has a variable type name of sat and key values SHIA and SFT; the qid is then "sat/SHIA/SFT".
obj narrative
Subclasses should override this method as needed. Returns a brief narrative description of the difference between val1 and val2.
obj trivial
Subclasses should override this method as needed. Returns 1 if the difference between val1 and val2 is trivial, and 0 otherwise. By default, this method simply returns 0; we wouldn't normally create a vardiff(n) instance unless the two values were known to be non-identical. Subclasses can override this method to provide an additional check, e.g., an epsilon test.
obj val1
Returns the raw value of the variable in the first scenario.
obj val2
Returns the raw value of the variable in the second scenario.
obj view
Returns a view dictionary for the vardiff, including the type, name, category, key values, and so forth.

PRIVATE INSTANCE COMMANDS

Each instance of vardiff(n) provides the following private subcommands:

obj AddInput vartype val1 val2 keys...
Given a vardiff(n) type, a pair of values, and the variable keys, checks whether the two values are non-trivially different and if so saves a vardiff object for them. This is intended to be used in each subclass's FindInputs method.
obj FindInputs
Subclasses should override this method as needed. Compares the inputs to the variable across the two scenarios, and adds possible differences using the private method AddInput. The differences are also added to the comparison(n) object. This is used when drilling down to provide an explanation of a particular difference.

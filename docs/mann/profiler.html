SYNOPSIS

package require projectlib 6.3.3
namespace import ::projectlib::*
  • profiler clear
  • profiler dump
  • profiler profile cmd ?subcmd...?
  • profiler reset
  • profiler unprofile cmd ?subcmd...?

DESCRIPTION

profiler(n) allows the developer to dynamically instrument routines of interest, and determine how many times they are being called, and for how long, as an aid to performance optimization. The results should be interpreted with care, as the instrumentation adds some performance overhead; e.g., if command A calls command B, and both are instrumented, A's run-time will include the cost of B's instrumentation.

Use profiler profile to instrument a procedure, Snit typemethod, or Snit instance method. Use profiler unprofile to de-instrument a procedure, or profiler reset to remove all instrumentation. Use profiler dump to dump the current measurements, and profiler clear to delete them in preparation for subsequent runs.

COMMANDS

This module defines the following commands:

profiler clear
Clears all measurements, leaving a clean slate.
profiler dump
Returns a dump of the current results. The dump includes, for each profiled command, the number of times it was called, and the total time in microseconds.
profiler profile cmd ?subcmd...?

Instruments the named command, which can be a procedure, a Snit typemethod, or a Snit instance method. For instance: 

    # Procedure MyProc
    profiler profile MyProc

    # Typemethod "assess"
    profiler profile ::mytype assess

    # Instance method "compute"
    profiler profile $object compute

Notice that when you profile an instance method, you're profiling that method for all instances of the type, not just the object named in the call.

profiler reset
Resets the package, removing all measured results and de-instrumenting all instrumented commands.
profiler unprofile cmd ?subcmd...?
Removes the instrumentation on the specified command, which is named as in profiler profile.

ENVIRONMENT

This package requires Tcl 8.5 or later.

To use the package in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.

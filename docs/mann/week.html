DESCRIPTION

The Athena simulation's time tick is one week. For convenience, we map this to real time in the form of Julian weeks. Each year consists of 52 Julian weeks, starting on January 1st. The week containing February 29th contains eight days during leap year, and the last week in December always contains eight days. Athena ignores these distinctions, just as it ignores the days of the week.

The week(n) type is responsible for converting between Julian week strings and the integer number of Julian weeks since the epoch, which for this type is defined as the first week of 2000 AD. Thus, the first week of 2000 AD is week 0, the first week of 2001 is week 52, the first week of 2002 is week 104, and so on. Note that negative week numbers are perfectly fine; the first week of 1999 is week -52.

Julian Week Strings

The string representation of a particular Julian week is a string yyyyWww where yyyy is the four digit year, and ww is the Julian week number for that year, 1 to 52. Thus, the first week of 2000 (the epoch) is represented as 2000W01, and the 37th week of 2012 is represented as 2012W37.

COMMANDS

This module defines the following commands:

week toString week
Converts an integer week expressed as the number of weeks since the epoch into a week string.
week toWeek ts
Converts a Unix timestamp to the integer number of weeks since the epoch. It could be a negative number.
week toInteger wstring
Converts a week string into the integer number of weeks since the epoch.
week toTimestamp wstring
Converts a week string into a standard Unix timestamp. Note that since Athena operates in time units of weeks, using week(n) to convert from a Unix timestamp and back will not result in the same timestamp being returned.
week validate wstring
Validates a week string. If the string is valid, it is returned in canonical form (i.e., with a capital "W"). If not, throws an error with error code INVALID.

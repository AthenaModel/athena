DESCRIPTION

This man page describes a component of the athena(n) package that provides access to the scenario's CURSE entities. A CURSE is a Complex User-defined Role-based Situation or Event, i.e., a magic attitude driver used to model events and situations not included in Athena's normal rule base.

The component is accessible as a subcommand of the athena(n) and athenadb(n) objects; Public Subcommands are accessible from athena(n) and (usually) from athenadb(n), and Private Subcommands only from athenadb(n).

COMMANDS

Public Subcommands

The adb curse command takes the following subcommands, where adb is an instance of athena(n).

adb curse check
Computes a sanity check for the set of curses, returning a list of two elements, {code flist}, where the code is OK or WARNING, and flist is a list of sanity check failure records. If the code is OK, the list of failures will be empty.
adb curse exists curse_id
Returns 1 if there is a CURSE called curse_id, and 0 otherwise.
adb curse get curse_id ?parm?
By default, returns a dictionary of information about CURSE curse_id, as read from fmt_curses. If parm is given, returns the value of the parm.
adb curse namedict
Returns an ID/longname dictionary of the CURSE names.
adb curse names
returns a list of the IDs of the CURSEs.
adb curse validate curse_id
Validates ID curse_id, returning curse_id if valid and throwing INVALID otherwise.

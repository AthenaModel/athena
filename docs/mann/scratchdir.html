scratchdir(n) -- Scratch Directory Utility, submodule of projectlib(n)

SYNOPSIS

package require projectlib 6.3.3
namespace import ::projectlib::*
  • scratchdir clear
  • scratchdir init ?scratchdir?
  • scratchdir join ?arg arg...?

DESCRIPTION

scratchdir(n) is responsible for locating and creating scratch directories for arachne(1) and other applications that require a well-known scratch directory.

At start up the application calls scratchdir init, optionally passing it a specific scratch directory name. By default, the scratch directory will be appdir/scratch/, where the appdir is defined as for appdir(n).

The scratch directory may be referenced by multiple applications at once, i.e., a master application such as arachne(1) and any number of ancillary tools, such as the athena_log(1) logbrowser. Only the master should write to the scratch directory.

Normally, the master application will initialize and clear the scratch directory: 

    scratchdir init
    scratchdir clear

The scratchdir clear command also creates the normal set of subdirectories, e.g., scratchdir/log.

COMMANDS

This module defines the following commands:

scratchdir clear
Deletes the content of the scratch directory, including all subdirectories, and creates the standard subdirectories. It's an error to call this command prior to scratchdir init.
scratchdir init ?scratchdir?

Initializes scratchdir(n). If scratchdir is given, it is normalized and saved as the scratch directory; otherwise, the scratch directory will be appdir/scratch/, where the appdir is defined as for appdir(n).

Successive calls to this command in one application are no-ops.

scratchdir join ?arg arg...?

Called with no arguments, returns the name of the scratch directory. Otherwise, returns the path name resulting from joining the arguments to the scratch directory. For example, the following command returns the absolute path of a file called "working.rdb" in the RDB directory. 

set rdbfile [scratchdir join rdb working.rdb]

SEE ALSO

workdir(n), which creates working directories for applications like Athena Workbench, athenawb(1), where only the application itself needs access to the directory.

DESCRIPTION

Each instance of athena(n) contains an executive interpreter for processing scripts and expressions; the interpreter's complete API is exposed via the athena(n) instance's executive subcommand.

The heart of the executive is a marsutil(n) smartinterp(n) object; many of the executive's commands are delegated to the smartinterp(n).

This man page documents the executive API.

INTERNAL SCRIPTS

The executive's script command is used to manage the scenario's collection of internal scripts. Any number of internal scripts may be saved using script save. A saved script can be executed on demand, using the call method or executive command; alternatively, a script that defines Tcl procedures can be loaded automaticaly on reset by setting its script auto flag.

Scripts can be used to automate tasks (i.e., the set up for particular excursions from a base case) or with EXECUTIVE tactics.

COMMANDS

executive(n) provides the following commands for use by the application.

executive obj adb_ ?options...?

Creates a new executive instance named obj, and returns the fully qualified object name. The adb_ is the name of the enclosing athenadb(n) object. The object takes the following options:

-executivecmd cmd
This option names a command to be called whenever the executive is reset. The cmd is a prefix to be called with one additional argument, the executive object itself. The application will usually use this to define additional application-specific commands for use in executive scripts.

INSTANCE COMMANDS

Each instance of executive(n) provides the following.

obj ensemble ...
obj expr ...
obj function ...
obj proc ...
obj smartalias ...
These commands are delegated to the underlying smartalias(n) interpreter; they are used to define new executive commands and functions. See the smartalias(n) man page for details.
obj call scriptName
Executes the named script in the interpreter. The scriptName should either be a script name as returned by script names, or an absolute or relative path to a file on the local disk.
obj check script
Checks a script for obvious syntax errors. Returns a flat list of line numbers and error messages.
obj commands
Returns a list of the commands defined in the interpreter.
obj errtrace
Returns the error stack trace from the last eval or call that threw an error.
obj eval script
Evaluates the script in the interpreter, returning the result. If the script throws an error, the stack trace is available by calling errtrace.
obj reset
Resets the executive interpreter, restoring it to a pristine state with all standard executive commands and functions defined. If there is a -executivecmd callback, it will be called when the reset is complete, allowing application commands to be defined in the interpreter.
obj script subcommand ?args...?

The client can save scripts into the scenario for later execution via the call method or executive command. This command's subcommands are used to manage the scenario's collection of scripts.

obj script names
Returns a list of the names of the scripts in sequence order.
obj script list
Returns a human-readable list of the names of the defined executive scripts in sequence order, one per line. The result is not a proper Tcl list if any of the names contain whitespace.
obj script get name
Returns the body of the named script.
obj script exists name
Returns 1 if there's a script with the given name, and 0 otherwise.
obj script load name
Loads the named script into the executive interpreter. No special error handling is done; it's assumed that the caller will handle any errors.
obj script save name ?body? ?-silent?

Saves a script called name into the scenario. If body is given, it's the script's new body; if not, the script is given a body consisting of a single header comment.

By default, the executive will send the <Scripts> event on save; if the -silent option is given, it will not. (See athena(n) for more about notifier events.)

obj script auto name ?flag?
Queries and sets the value of the named script's "auto" flag, which is false by default. If true, the script will be loaded automatically on executive reset.
obj script import filename
Imports a script from the named file into the scenario; a name is assigned automatically.
obj script rename oldName newName
Renames a script.
obj script sequence name priority
Scripts are saved in priority order; on reset, all scripts with their script auto flag set are script loaded in priority order. This command is used to move the named script in the sequence. The priority can be one of top, raise, lower, or bottom.

ENVIRONMENT

Requires Tcl 8.5 or later.

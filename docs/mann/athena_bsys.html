DESCRIPTION

This man page describes a component of the athena(n) package that gives access to the scenario's belief systems.

The component is accessible as a subcommand of the athena(n) and athenadb(n) objects; Public Subcommands are accessible from athena(n) and (usually) from athenadb(n), and Private Subcommands only from athenadb(n).

COMMANDS

Public Subcommands

The adb bsys command takes the following subcommands, where adb is an instance of athena(n).

adb bsys affinity sid1 sid2
Returns the belief system sid1's affinity for belief system sid2. Affinity is a number between -1.0 and 1.0.
adb bsys belief subcommand ?args?

This command is used to query the beliefs in a belief system. The subcommands are as follows:

adb bsys belief cget option

Returns the value of the named option. Options include:

-position number
Retrieves the system's position with respect to the topic, a real number from -1.0 to +1.0. Defaults to 0.0. The client can use qposition to format position values.
-emphasis number
Retrieves the system's emphasis on agreement or disagreement with respect to this topic, where 0.0 represents a complete emphasis on disagreement and 1.0 represents a complete emphasis on agreement. The default is 0.5. The client can use qemphasis to format emphasis values.
adb bsys belief view sid tid
Returns a dictionary of information about system sid's belief about topic tid.
adb bsys belief validate bid
Validates bid as an {sid tid} pair for this scenario. Returns the bid, or throws INVALID on failure.
adb bsys belief isdefault bid
Returns 1 if belief bid has its default value, and 0 otherwise, where bid is a valid {sid tid} pair.
adb bsys playbox subcommand ?args?

This command is used to query information about the beliefs of the entire playbox.

adb bsys playbox cget option

Returns the value of the named option. Options include:

-gamma value
This is the playbox "commonality" dial; it controls the degree of implicit commonality among the entities in the playbox, that is, the number of implicit topics about which there is general (and passionate) agreement in the playbox. If -gamma is 0, there is no implicit commonality; only explicit topics are considered. If -gamma is 1.0 there are as many implicit topics as explicit topics; if 0.5, half as many; if 2.0, twice as many. The value must be no less than 0.0; default is 1.0.
adb bsys playbox view
Returns a dictionary of the playbox data with the values formatted for display.
adb bsys system subcommand ?args?

This command is used to query information about entire belief systems.

adb bsys system cget sid option

Retrieves the value of the given option for belief system sid. Valid options are as follows:

-name name
This is the name of the belief system, for display to the user. Names must be unique across belief systems. The default name is "System sid".
-commonality value
This is the belief system "commonality" dial; it specifies the extent of the system's participation in the dominant culture in the playbox. If 1.0, the system shares all of the implicit cultural commonality; if 0.0, none of it. Defaults to 1.0. Note that the effect of this option depends on the playbox -gamma value; see playbox cget.
adb bsys system exists sid
Returns 1 if the belief system exists, and 0 otherwise.
adb bsys system ids
Returns a list of the IDs of the defined belief systems.
adb bsys system inuse sid
Returns 1 if the given belief system is assigned to at least one actor or group, and 0 otherwise.
adb bsys system validate sid
Returns sid if it is a valid belief system ID, and throws INVALID otherwise.
adb bsys system view sid
Returns a dictionary of the system's data with the values formatted for display.
adb bsys topic subcommand ?args?
adb bsys topic cget tid option

Retrieves the value of the given option for belief topic tid. Valid options are as follows:

-name name
This is the name of the belief topic, for display to the user. Names must be unique across belief topics. The default name is "Topic tid".
-affinity flag
Specifies a numeric flag, 0 or 1. If 1 (the default), then beliefs on this topic will be included in the computation of affinity; if 0, then it will be excluded.
-relevance fraction
Specifies the relevance of the topic as a fraction from 0.0 to 1.0 (the default). Positions on a topic have decreasing effect as the topic's relevance declines.
adb bsys topic exists tid
Returns 1 if the belief topic exists, and 0 otherwise.
adb bsys topic ids
Returns a list of the IDs of the defined belief topics.
adb bsys topic inuse tid
Returns 1 if the given belief topic is assigned to at least one semantic hook, and 0 otherwise. See athena_hook(n).
adb bsys topic validate tid
Returns tid if it is a valid belief topic ID, and throws INVALID otherwise.
adb bsys topic view tid
Returns a dictionary of the topic's data with the values formatted for display.

DESCRIPTION

This man page describes a component of the athena(n) package used to automatically generate entire scenarios or parts of scenarios as an aid to testing and optimization.

The component is accessible as a subcommand of the athena(n) and athenadb(n) objects; Public Subcommands are accessible from athena(n) and (usually) from athenadb(n), and Private Subcommands only from athenadb(n).

COMMANDS

Public Subcommands

The adb autogen command takes the following subcommands, where adb is an instance of athena(n).

adb autogen scenario options...

Creates a scenario with a default number of each of Athena's entities or, if any of the options are supplied, the number of entities requested by the user. If the -nostrategy flag is supplied then no tactics are created, otherwise a default set of tactics are created for each actor.

The available options are:

-nb n
Specifies the number of neighborhoods to create. Each neighborhood is a 10x10 pixel square. Neighborhoods are laid out in strips starting at the top of the map. Defaults to 10.
-actors n
Specifies the number of actors to create. Defaults to 3.
-frcg n
Specifies the number of force groups to create. As they are created, the force group type is cycled through each possible type. Each force group is given a base personnel of 100000. Defaults to 3.
-civg n
Specifies the number of civilian groups to create per neighborhood. The last group in each neighborhood is created with the subsistence agriculture flag set to true. The different types of housing are cycled through as each group is created. The exception to this are subsistence agriculture groups, which are always AT_HOME. The population of every group in a neighborhood is some non-negative multiple of 10000. Thus, the first group in a neighborhood has a base population of zero and each group created after has a base population of 10000 more than the previous group. Defaults to 6.
-orgg n
Specifies the number of organization groups to create. As groups are created, the organization type is cycled through. Each group has a base personnel of 100000. Defaults to 2.
-topics n

Specifies the number of belief system topics to create. Each actor and civilian group is given its own belief system, and assigned a belief for each topic. The beliefs assigned are cycled through four possibilities comprised of a position and emphasis pair. They are:

  • Passionately For and Agreement--Strong
  • Passionately For and Disagreement--Extreme
  • Passionately Against and Agreement--Strong
  • Passionately Against and Disagreement--Extreme

Note that if 4 topics are requested then all actors and civilians will have identical belief systems resulting in affinities of 1 between all entity pairs. Defaults to 3.

-nostrategy
If specified, no tactics are employed by any actor.

To create a scenario with 8 neighborhoods and the default number of entities for all others, enter 

    autogen scenario -nb 8

To create a scenario with no tactics for any actor, enter 

    autogen scenario -nostrategy
adb autogen actors ?num?
Automatically generates the requested number of actors provided that no actors already exist. If no argument is supplied, three actors are created.
adb autogen nbhoods ?num?

Automatically generates the requested number of neighborhoods subject to certain constraints. Each neighborhood is a 10x10 pixel square. Neighborhoods are laid out in strips starting at the top of the map. If no argument is supplied, ten neighborhoods are created. The constraints are as follows:

  • No neighborhoods exist
  • At least one actor exists
adb autogen civgroups ?num?

Automatically generates the requested number of civilian groups per neighborhood. The last group in each neighborhood is created with the subsistence agriculture flag set to true. The different types of housing are cycled through as each group is created. The exception to this are subsistence agriculture groups, which are always AT_HOME. The population of every group in a neighborhood is some non-negative multiple of 10000. Thus, the first group in a neighborhood has a base population of zero and each group created after has a base population of 10000 more than the previous group. If no argument is supplied, six groups per neighborhood are created.

The creation of civilian groups is subject to the following constraints:

  • Neighborhoods must exist
  • No civilian groups exist
adb autogen frcgroups ?num?

Specifies the number of force groups to create. As they are created, force group type is cycled through each possible type. Each force group is given a base personnel of 100000. If no argument is supplied, three groups are created.

The creation of force groups is subject to the following constraints:

  • At least one actor must exist
  • No force groups exist
adb autogen orggroups ?num?

Specifies the number of organization groups to create. As they are created, organization group type is cycled through each possible type. Each group is given a base personnel of 100000. If no argument is supplied, three groups are created.

The creation of organization groups is subject to the following constraints:

  • At least one actor must exist
  • No force groups exist
adb autogen bsystem ?num?

Specifies the number of belief system topics to create. Each actor and civilian group is given its own belief system, and is assigned a belief for each topic. The beliefs assigned are cycled through four possibilities comprised of a position and emphasis pair. They are:

  • Passionately For and Agreement--Strong
  • Passionately For and Disagreement--Extreme
  • Passionately Against and Agreement--Strong
  • Passionately Against and Disagreement--Extreme

If no argument is supplied, three topics are created.

Creation of a belief system is subject to the following constraints. The constraints are as follows:

  • At least one actor must exist
  • Civilian groups must exist
  • No topics exist
adb autogen assign ?options?

This command automatically generates one or more ASSIGN tactics for force or organization groups that belong to the specified owner. By default all activities are assigned in turn for each neighborhood. For each tactic created, 100 personnel are specified to be doing the activity. The list of available options are:

-frcg glist
List of force groups owned by owner that should be assigned activities. Defaults to all force groups owned by the owner.
-orgg glist
List of organization groups owned by owner that should be assigned activities. Defaults to all organization groups owned by owner.
-nbhoods nlist
List of neighborhoods that activities should be assigned in. Defaults to all neighborhoods.
-frcact alist
List of force activities that should be assigned, in turn, to each force group in each neighborhood. Defaults to all force activities.
-orgact alist
List of organization activities that should be assigned, in turn, to each organization group in each neighborhood. Defaults to all organization activities.

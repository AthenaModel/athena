DESCRIPTION

This module defines the ::projectgui::sorter widget, which allows the user to sort a list of items into bins. The widget implements most of the field(i) interface, with the exception of the -state option.

The use case is as follows. The application has a set of items (e.g., human-readable reports from some external source) and needs the user's help to sort them into bins to enable further automated processing. The application sets up the sorter(n) widget with the set of bins, a list of unique IDs for the items to sort, and the means to retrieve information about the items. The user sorts the items into bins, and then returns control to the application to use the sorted data.

Configuration

A sorter(n) widget contains three panes: the bin set, the item list, and the detail pane. The bin set is a collection of bin buttons; it always contains at least two bins, unsorted and ignored. Items start out in the unsorted bin. The user may put items that don't fit any of the other bins into the ignored bin.

The bin set is configured using the -bincols and -binspec options.

The item list is a databrowser(n) widget; it is configured using the -itemcmd and -itemlayout options.

The detail pane is an htmlviewer(n); it is configured using the -detailcmd and -helptext options.

COMMANDS

This module defines the following command:

sorter win ?options...?

Creates a new sorter widget named win.

The sorter widget may be created with the following options:

-bincols num
Bins can be layed out in 1, 2, or more columns. The default is 2.
-binspec dict
The "bin specification" is a dictionary of bin ID strings and text labels that specifies the bins into which items can be sorted. The IDs should be short ID strings containing no whitespace or special characters. The labels are displayed to the user.
-changecmd cmd
The cmd is called each time the value of the widget changes; the new value is passed to the command as an additional argument. The value of the widget is a dictionary whose keys are the bin IDs (including unsorted and ignored) and whose values are lists of item IDs in the bins.
-detailcmd cmd
This command is called when an item is selected in the item list; it is passed the selected item's ID as an additional argument. It should return an HTML string that gives all of the item's details, for display in the detail pane.
-helptext html
This option specifies HTML text to be displayed in the detail pane when no item is selected in the item list. Usually this text will explain what the items are, what the bins mean, and how to use the widget.
-itemcmd cmd
This command is called by the item list to retrieve a dictionary of information about each item in the list. The keys should match the column names in the -itemlayout specification. It is mapped to the databrowser(n)'s -dictcmd option.
-itemlabel text
The sorter(n) widget can sort all kinds of items. The text is a plural noun indicating the kind of items being sorted; it defaults to "Items".
-itemlayout spec
This option defines the column layout for the itemlist. It is mapped to the databrowser(n)'s -layout option, and has the identical syntax.

INSTANCE COMMAND

win cget option
Returns the value of the specified option.
win configure option value...
Sets the values of the specified options.
win clear
Clears the widget of all item data, and calls the -changecmd.
win get ?bin?

By default, returns the widget's value, which is a bindict: a dictionary whose keys are bin IDs (including unsorted and ignored) and whose values are lists of item IDs.

If a bin ID is given, returns the list of item IDs currently in that bin. Note that a single item can be sorted into multiple bins.

win set bindict
Sets the widget's value to given bindict, where the bindict is as described for the get subcommand. Calls the -changecmd.
win sortset idlist
Clears the widget and puts the IDs in the idlist into the unsorted bin.

ENVIRONMENT

sorter(n) requires Tcl 8.5 or later.

DESCRIPTION

helpserver(n) is a mydomain(i) implementation designed for browsing help(5) help files. The application creates an instance of helpserver(n), specifying a help database using the -helpdb option, and registers the instance with myagent(n). At that point, myagent(n) clients like mybrowser(n) can query it.

The resources known to the server are listed under RESOURCES; they can also be seen by querying the server's /domain/urlhelp resource.

Searching

A page URL can include a query in the usual way, delimited with a question mark, e.g., 

    /domain/?my search text

A query can be added to any page URL; however, the page path is discarded and doesn't affect the search.

Searching proceeds as follows:

  1. If the search text exactly matches a page title, that page is returned. This match is case-insensitive.
  2. Otherwise, the search text is passed to SQLite3's full text search engine, and a page of links to matched pages is returned.

CONTENT TYPES

All help(5) pages have the content type text/html. Embedded images have the content type tk/image. All page URLs can also be retrieved as tcl/linkdict, in which the data returned is a dictionary of links to the page's children, if any.

RESOURCES

In addition to the standard resources provided by mydomain(n), helpserver(n) provides the following:

/domain/

This is the main help page; lower level pages can be referenced by adding on to the path in the usual way. The text/html content type is the help text itself; the tcl/linkdict content type is a dictionary of links to the child pages, if any.

If the URL contains a query, the help database will be searched. See Searching for details.

/domain/image/name
Returns the tk/image resources compiled into the help database with the given name.

COMMANDS

helpserver(n) provides the following commands for use by the application.

helpserver name ?options...?

Creates a new instance of helpserver(n) called name; returns the fully-qualified command name.

helpserver(n) has the following option:

-domain domain
The help server's folder in the application's URL tree. It must begin with a "/" and consist of a single folder name. Defaults to "/help". All help resources are accessed at URLs beginning with this name.
-helpdb database
Specifies the help(5) database to query, expressed as an sqldocument(n) object.
-headercmd command
Specifies a command that will be called for each returned text/html help page. It take one argument, the "udict" passed to the mydomain(n) content handler, and returns a snippet of HTML text to be inserted at the top of the HTML page.

INSTANCE COMMANDS

Instances of helpserver(n) provide the following subcommands for use by the application.

$server ctypes rtype
Given a resource type name, rtype, returns a list of the available content types.
$server get url ?contentTypes?

From mydomain(i): Retrieves the url from the server. If the url cannot be retrieved for any reason, the server throws an error with error code NOTFOUND; the error message is a human-readable message explaining why the resource cannot be found.

The contentTypes argument, if given, is a list of content types accepted by the client. Browse the server's /urlhelp resource to see what URLs are accepted, and what content types they can provide. Custom content types should be documented by the application.

If contentTypes is empty, the server will return the first available content type for the requested url. Otherwise, it will return the first content type listed in contentTypes which is available for the requested url.

Note that the content types listed in contentTypes can include wild cards, e.g., text/* and */*.

On success, the get command returns a dictionary with the following keys:

url
The complete URL given to the server. This is of interest because the myagent(n) might have performed address resolution; hence, the URL given to the server might not be the same as the URL given to the agent.
contentType
The content type of the returned content.
content
The returned content itself.
$server resources
Returns a list of the names of the registered resource types.

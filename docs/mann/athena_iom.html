DESCRIPTION

This man page describes a component of the athena(n) package that provides access to the scenario's information operation messages (IOMs). An IOM represents a message tailored to affect the attitudes of a particular part of the population. Its success depends on its semantic hook (athena_hook(n)), and its effects are based upon its payloads (athena_payload(n)).

The component is accessible as a subcommand of the athena(n) and athenadb(n) objects; Public Subcommands are accessible from athena(n) and (usually) from athenadb(n), and Private Subcommands only from athenadb(n).

COMMANDS

Public Subcommands

The adb iom command takes the following subcommands, where adb is an instance of athena(n).

adb iom check
Computes a sanity check for the set of IOMs and returns a list of two items, {severity flist}, where severity is either OK or WARNING and flist is either the empty list or a list sanity check failure dictionaries.
adb iom exists iom_id
Returns 1 if there is an IOM called iom_id, and 0 otherwise.
adb iom get iom_id ?parm?
By default, returns a dictionary of information about IOM iom_id, as read from the fmt_ioms. If parm is given, returns the value of the parm.
adb iom normal subcommand ?args...?

Returns information about the "normal" IOMs, i.e., those whose state is normal as opposed to disabled or invalid.

adb iom normal namedict
Returns an ID/longname dictionary of the names of the "normal" IOMs.
adb iom normal names
Returns a list of the IDs of the "normal" IOMs.
adb iom normal longnames
Returns a list of the "long names" of the "normal" IOMs.
adb iom normal validate iom_id
Validates IOM iom_id, returning iom_id if iom_id is a "normal" IOM and throwing INVALID otherwise.
adb iom namedict
Returns an ID/longname dictionary of the IOM names.
adb iom names
Returns a list of the IDs of the IOMs.
adb iom validate iom_id
Validates group ID iom_id, returning iom_id if valid and throwing INVALID otherwise.

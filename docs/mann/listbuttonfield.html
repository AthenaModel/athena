DESCRIPTION

The listbuttonfield(n) widget is used for the same purpose as the listfield(n) widget, to select one or more items from a list of items; however, it is designed for use when space is at a premium. The widget displays a label and an "Edit" button; the label displays the chosen value, and the "Edit" button pops up a messagebox(n) in which the user can select the desired items.

COMMANDS

This module defines the following commands.

listbuttonfield pathName ?options...?

Creates a new listbuttonfield widget named pathName, and returns pathName. The widget supports the following options, in addition to those provided by the underlying frame widget:

-changecmd command
From field(i). Specifies a command to be called whenever the field's value changes, for any reason whatsoever (including explicit calls to set. The new value is appended to the command as an argument.
-emptymessage text
The text to display in the field when no items are selected. Defaults to "No items selected."
-itemdict dict
The items from which the user can choose, specified as a dictionary of keys and values. The dictionary values are displayed in the dialog's "include" and "omit" lists, optionally prefixed by the keys (see -showkeys); the get and set methods return and accept a list of keys.
-message text
Specifies the message to be shown at the top of the pop-up dialog. Defaults to "Select items from the list:".
-listrows rows
The height of the "include" and "omit" lists, in rows, in the dialog.
-listwidth chars
The width in text characters of each of the item lists.
-showkeys flag
If true (the default), the text shown for each item in the dialog's item lists is "key: value". Otherwise only the -itemdict values are displayed.
-showmaxitems num
Specifies the maximum number of list items to show in the narrative box; defaults to 10. If num is the empty string, there is no limit.
-state state
From field(i). listfield(i) widgets have states, just as the normal tk:: and ttk:: widgets do. The valid states are normal and disabled.
-stripe flag
If true (the default) the even items in each list in the dialog are striped with a distinctive background color. This is useful if item values might wrap to multiple lines.
-wraplength pixels
Specifies the width in pixels at which the widget will wrap the narrative across multiple lines. Defaults to 300.
-width chars
Specifies the width of the widget's narrative box in characters. If negative, specifies the minimum width. Defaults to -30.

INSTANCE COMMAND

listbuttonfield(n) widgets have the following subcommands:

pathName cget option
Returns the value of the specified option.
pathName configure option value...
Sets the value of one or more of the object's options.
pathName get
Returns the current value of the field, i.e., a list of keys from the -itemdict.
pathName set value
Sets the current value of the field, a list of keys from the -itemdict.

FIELD TYPE

projectlib(n) defines a dynaform_field(i) field type called listbutton. The field type takes the same -listrows, -listwidth, -message, -showmaxitems, -showkeys, -stripe and -wraplength options as listbuttonfield(n) does. The -width option sets the minimum width of the field, since that's usually what's wanted.

The widget's itemdict is set using either -dict or -dictcmd, as for the enumlist field type; one of these two options is required.

ENVIRONMENT

projectgui(n) requires Tcl 8.5 or later.

To use projectgui(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.

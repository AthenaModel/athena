DESCRIPTION

parmdict(n) is a tool for managing and validating parameter dictionaries, e.g., the query dictionary in an HTTP request. A handler can "prepare" each of the parameter values, possibly transforming or validating them along the way. If there are errors, the errors can be reported back.

In general, parmdict(n) provides an order(n)-like validation system for use in general procedures.

COMMANDS

::projectlib::parmdict is an oo::class. It provides the following commands.

parmdict create name ?dict
Creates a new instance of parmdict(n) with the given command name for the given initial parameter dict.
parmdict new ?dict?
Creates a new instance of parmdict(n) for the given initial parameter dict and returns the instance's name.

INSTANCE COMMANDS

Instances of parmdict(n) provide the following subcommands.

obj assign parm ?parm...?
Assigns each listed parm's value to a variable of the same name in the caller's context. It's an error if parm isn't defined in the parmdict, so be sure to prepare.
obj badparm parm
Returns 1 if it's known that the parm requires no further validation: either there is already a known error, or the parameter is optional and its value is the empty string.
obj checkon parm script
This command is used when prepare's options aren't enough to do the job of validation, e.g., when two parameters must be validated together. If the parm isn't already known to be "bad" (in the sense of badparm), the script will be executed in the caller's context. If the script throws INVALID, the error will be caught and parm will be rejected. Naturally, reject and other commands can also be used explicitly.
obj errors

If parameter errors have been discovered, returns a dictionary of parameter names and error messages. In some cases, the error dictionary might have one key, "*", and a message encompassing the whole set of parameters.

If no errors have been discovered, this command returns an empty dictionary.

obj get parm
Returns the value of parm if parm is defined, and "" otherwise.
obj ok
Returns 1 if no parameter errors have been discovered, and 0 otherwise.
obj parms
Returns the current parameter dictionary.
obj prepare parm ?options...?

This one command does most of the work. First it ensures that the parm is present in the dictionary, adding it with an empty value if necessary. Next it executes each of the options in order from left to right; each option can transform or validate the parameter's value in some way. The command returns the final value of the parameter.

The options are as follows:

-default defvalue
If the parameter's value is the empty string, assigns defvalue.
-ident
Requires that the parameter's value be a valid identifier, as determined by the identifier type in marsmisc(n).
-notin list
Requires that the parameter's value not be in the given list; if it is, it will be rejected with the message "Duplicate parm".
-remove
Removes the parameter from the dictionary. When this option is used, the caller will usually capture the command's return value.
-required
The parm is required; it is rejected if its value is the empty string.
-toupper
Converts the parameter's value to upper case.
-tolower
Converts the parameter's value to lower case.
-type vtype
Validates the parameter's value using the named validation type, saving the return value. If the command throws INVALID, the parameter will be rejected. Note that "-type mytype" is equivalent to "-with {mytype validate}.
-with cmd
Passes the parameter's value to the cmd as an additional argument, saving the return value. If the command throws INVALID, the parameter will be rejected.
obj prune
Removes any parameters for which prepare has not been called.
obj reject parm message
A client calls this to explicitly reject a particular parm's value. The message is added to the errors dictionary.
obj remove parm
Removes the parameter from the parmdict, returning its value. If the parameter is undefined, returns "".
obj setdict dict
Re-initializes the object with the new parameter dict.

DESCRIPTION

enumx(n) is a replacement for the marsutil(n) enum(n) enumerated value definition type, implemented using TclOO instead of Snit. Each instance of the class defines an enumeration type: an ordered set of symbolic names, possibly with equivalent forms. An enumx(n) type allows symbolic names to be validated, and translated to their numeric indices and to equivalent forms.

An enumx(n) type can define any number of equivalent forms.

Differences from enum(n)

This type differs from enum(n) in the following ways:

  • Short names can be translated into indices and equivalent forms, but not vice versa. As enum(n) types were originally used, the short names, long names, and indices were all equally valid in input, but we haven't made use of that in a long time.
  • The API has been streamlined; rarely used subcommands have been eliminated.
  • A type can define any number of equivalent forms, rather than just one kind, the long name.
  • Because this type is implemented using TclOO, it's easy to add type-specific subcommands using oo::objdefine. (This was the original motivation for defining this new type.)

Customization

Because an enumx(type) is a TclOO object, new subcommands can be added to it as object methods using oo::objdefine. For convenience, the type constructor takes an oo::objdefine script as an optional argument.

COMMANDS

enumx create name defdict ?custom?

enumx create defines a new enumeration definition type called name, which becomes a Tcl command defined in the current scope; enumx create returns the fully-qualified form of name.

If custom is given, it is an oo::objdefine script used to add type-specific subcommands to the new type.

The symbolic names and any equivalents are given by the defdict, which has the form: 

name {form value ...}
name {form value ...}

For example, the following enumeration has two equivalent forms, longname and grade: 

enumx create MyEnum {
    G  {longname "The Good" grade A}
    B  {longname "The Bad"  grade C}
    U  {longname "The Ugly" grade F}
}

Alternatively, the defdict can be the empty string, "", and values can be added later using the add subcommand.

A valid enumeration value is a symbolic name, regardless of case. The canonical form of the value has the case used in the defdict.

Note: It usual for all symbolic names to have the same equivalent forms, but this is not required.

INSTANCE COMMAND

Each enumx(n) type has the following subcommands (and may add others by subclassing or via oo::objdefine):

type add defdict
This command can add additional symbols to the enumeration, additional equivalent forms to existing symbols, and any combination of the two. Thus, the enumeration can be defined a little at a time. The syntax of defdict is the same as for enumx create.
type as form name
Returns the requested equivalent form of the symbolic name, or "" if it has none.
type asdict form
Returns a dictionary of symbolic names and equivalent values of the given form. If a given symbolic name lacks the requested form, the dictionary will contain the empty string for that name.
type aslist form
Returns a list of equivalent values of the given form, one for each symbolic name in index order. If a given symbolic name lacks the requested form, the list will contain the empty string for that name.
type defdict
Returns the enum's current definition dictionary, suitable for creating another enumeration.
type forms
Returns a list of the names of the equivalent forms defined by this type.
type html
Returns an HTML snippet which documents the enumeration's names and equivalent forms, suitable for including in a man page.
type index name
Returns the index of the given short name. If the name is not a valid short name, returns -1. Name lookups are case-insensitive.
type names
Returns a list of the symbolic names, in index order.
type validate name
Ensures that name is a valid symbolic name (ignoring case). If it is, it is returned with its canonical case. If it is not, an error is thrown with an -errorcode of INVALID.
type size
Returns the number of symbolic names in the enumerated type. Given n symbols, the symbols are indexed from 0 to n-1.

ENVIRONMENT

enumx(n) requires Tcl 8.5 or later, plus TclOO.

To use enumx(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the projectlib(n) library directory.

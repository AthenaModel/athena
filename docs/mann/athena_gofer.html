DESCRIPTION

This man page describes a component of the athena(n) package that allows the user to select and retrieve specific kinds of values according to rules.

A gofer value is a member of a gofer type that represents a particular data value or set of values to be returned on demand. This component defines a set of commands used to create, validate, and evaluate gofer values.

Each gofer type has a type name, e.g., CIVGROUPS and defines one or more rules for retrieving data of that type, each of which has a rule name, e.g., BY_VALUE.

A gofer value, then, is a dictionary which contains at least two keys, _type and _rule, which are set to the gofer type name and rule name respectively. The number and names of the remaining keys are type and rule-specific.

The complete set of gofer types defined by athena(n) may be found in the Athena on-line help.

The component is accessible as a subcommand of the athena(n) and athenadb(n) objects; Public Subcommands are accessible from athena(n) and (usually) from athenadb(n), and Private Subcommands only from athenadb(n).

COMMANDS

Public Subcommands

The adb gofer command takes the following subcommands, where adb is an instance of athena(n).

adb gofer eval gdict
Evaluates gofer value gdict, returning the requested data according to the type and rule.
adb gofer get typename rulename ?args...?
Makes a new gofer value with the given typename, rulename and argument list, and immediately evaluates it, returning the result.
adb gofer gtype typename
Returns the Tcl object that implements the named gofer type.
adb gofer keys typename
Returns the gofer type's keys, i.e., the list of parameter names defined by the type's rules.
adb gofer make typename rulename ?args...?
Makes a new gofer value with the given typename, rulename and argument list. The arguments must match the rule's parameter list.
adb gofer narrative gdict ?-brief?
Returns a narrative string for gofer value gdict. The narrative describes the rule for retrieving the data in human-readable terms, e.g., e.g., "all civilian groups resident in neighborhood NB1". If the -brief option is included, long lists in the narrative might be abbreviated with an ellipsis ("...").
adb gofer validate gdict
Validates gofer value gdict, returning it in canonical form if valid and throwing INVALID otherwise.
adb gofer typename subcommand args...

This command passes the subcommand and args along to the gofer type's Tcl object. This is primarily useful when working with a gofer value that is supposed to be of a specific type, e.g., 

# Validate a value of the CIVGROUPS gofer type

adb gofer CIVGROUPS validate gdict

The full set of gofer type subcommands is beyond the scope of this man page; see lib/athena/gofer_type.tcl.

Private Subcommands

The adb gofer command takes the following subcommands, where where adb is an instance of athenadb(n).

adb gofer check
Does a sanity check on all defined gofer types. This is provided for use by the test suite.

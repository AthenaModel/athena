SYNOPSIS

package require athena 6.3.3
namespace import ::athena::*
  • adb export fromcif scriptFile ?mapflag?
  • adb export fromdat scriptFile ?mapflag?

DESCRIPTION

This man page describes a component of the athena(n) package that exports the scenario as an order script.

The component is accessible as a subcommand of the athena(n) and athenadb(n) objects; Public Subcommands are accessible from athena(n) and (usually) from athenadb(n), and Private Subcommands only from athenadb(n).

COMMANDS

Public Subcommands

The adb export command takes the following subcommands, where adb is an instance of athena(n).

adb export fromcif scriptFile ?mapflag?

Creates an executive(n) script of send commands from the scenario's order history. (For historical reasons, the order history is known as the "CIF", or "Critical Input File".) The script will be written to the named scriptFile.

By default the order script will define the bounds of the playbox using a MAP:GEOREF order but will not contain any map image data. If the mapflag is true, then the full map image will be included.

NOTE: This command is provided as a way to capture the order history that led to the current scenario, which is useful under rare circumstances. For normal use, fromdata is preferred.

adb export fromdat scriptFile ?mapflag?

Creates an executive(n) script that will rebuild the current scenario using a minimal number of commands, i.e., unlike fromcif it does not recapitulate the entire order history. The script will be written to the named scriptFile.

By default the script will define the bounds of the playbox using a MAP:GEOREF order but will not contain any map image data. If the mapflag is true, then the full map image will be included.

Private Subcommands

None defined.

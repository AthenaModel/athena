DESCRIPTION

compdb(n) manages the Athena scenario comparison parameter database: the collection of parameters that govern how scenarios are compared, and how large differences have to be to be considered significant. At base, the parameter database is a parmset(n) with some added capabilities.

The full set of parameters is defined in compdb(5).

COMMANDS

compdb(n) provides the following subcommands for use by the application, along with all of the subcommands provided by the underlying parmset(n).

compdb init
Initializes the module. This should be called once, at application start-up. It defines the full set of comparison parameters.
compdb validate parm
Throws INVALID if parm isn't a defined parameter; otherwise returns parameter name in canonical form.
compdb nondefaults ?pattern?
Returns a list of the names of the parameters with non-default values. If pattern is given, it is a glob-style wildcard pattern; only names that match the pattern are included in the result.
compdb get parm
compdb names ?pattern?
compdb manlinks
compdb manpage
These commands are simply delegated to the underlying parmset(n).
compdb set parm value
Sets the parameter's value and saves the compdb to disk.
compdb reset
Resets all comparison parameters to their default values, and saves them to disk.
compdb list ?pattern?
Lists all parameters and their values. If pattern is given, it's a wildcard pattern; only parameters whose names match the pattern are listed.
compdb load
Loads the compdb from disk, if the compdb file exists. This is usually only done at application start-up.
compdb save filename
Saves any changes parameters to the file specified. No defaulted values are written, since that's unnecessary.

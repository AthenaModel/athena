DESCRIPTION

The rolemapfield(n) widget is used to associate lists of entities with user-defined roles in tactic dialogs, e.g., the CURSE tactic. When the user defines a "curse" entity with attitude injects, the groups and actors affected by the injects are specified using "roles": essentially, named arguments. When the CURSE tactic is set up to inject a given curse, the user needs to map those roles to actual lists of groups and actors.

The number and name of these roles will vary from curse to curse. However, the handling is identical for each role:

  1. There is a list of valid choices for each role, determined on-the-fly.
  2. The user may select multiple items from that list.
  3. The mapping is incomplete if any role's mapping is empty.

The rolemapfield(n) widget handles this use case. It is a thin wrapper around the form(n) widget that displays a configurable set of labels and edit fields given a -rolespec, which is a dictionary with this form:

role-name valid-list ...

The value of the widget is a similar dictionary,

role-name selected-list ...

Implementation

When configured with a -rolespec, the widget populates its form(n) widget with a label and a textfield(n) widget for each role. Each text field is read-only with an "Edit" button; displays a whitespace-delimited list of the currently chosen entities for that role. Pressing the "Edit" button pops up a messagebox(n) list selector.

COMMANDS

This module defines the following commands.

rolemapfield pathName ?options...?

Creates a new rolemapfield widget named pathName, and returns pathName. The widget supports the following options, in addition to those provided by the underlying frame widget:

-changecmd command
Specifies a command to be called whenever the selected entity changes, for any reason whatsoever (including explicit calls to set. The newly selected entity's name is appended to the command as an argument.
-listheight num
-liststripe flag
-listwidth num
These options are used to configure the messagebox(n) list selector when a role's mapping is being edited.
-rolespec spec
Sets the dictionary of roles and valid lists, and reconfigures the widget. See DESCRIPTION.
-state state
Sets the field's state to normal or disabled.
-textwidth num
The width of the role text fields, in characters.

INSTANCE COMMAND

rolemapfield(n) widgets have the following subcommands:

pathName cget option
Returns the value of the specified option.
pathName configure option value...
Sets the value of one or more of the object's options.
pathName get
Returns the current value of the field.
pathName set value
Sets the current value of the field.

ENVIRONMENT

projectgui(n) requires Tcl 8.5 or later.

To use projectgui(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.

DESCRIPTION

The wizman(n) widget is used to implement "wizard" interfaces. A wizard consists of a series of pages; the flow from one page to another is determined by the inputs on each page. Any kind of widget can be used as a page, provided that it adheres to the wizpage(i) interface.

COMMANDS

This module defines the following commands.

wizman pathName ?options...?

Creates a new wizman widget named pathName, and returns pathName. The widget is based on the ttk::notebook widget, and supports the same options. In addition, it defines the following options:

-cancelcmd command
Specifies a command to call when the user presses the "Cancel" button. The client should always set this option. The command should clean up after the wizard.
-finishcmd command
Specifies a command to call when the user reaches the final wizard page and presses the "Finish" button. The client should always set this option. The command should carry out the desired activity, and clean up after the wizard.

INSTANCE COMMAND

wizman(n) widgets have the following subcommands:

pathName cget option
Returns the value of the specified option.
pathName configure option value...
Sets the value of one or more of the object's options.
pathName add page ?options...?

Adds a wizpage(i) widget called page as a wizard page. The options are as for ttk::notebook tabs. By default, pages get the following layout options: 

-sticky nsew -padding 2
pathName nextpage
Returns the widget name of the next wizard page to go to.
pathName refresh
The wizman(n) widget must manage the navigation buttons that move the wizard from page to page, and enable or disable them based on the state of the currently displayed page. This function does this work; it should be called by the current page, either directly or indirectly, whenever the page's state has changed significantly.
pathName start
Starts the wizard going, displaying the first page and calling its enter command.
pathName thispage
Returns the widget name of the currently displayed wizard page.

ENVIRONMENT

projectgui(n) requires Tcl 8.6 or later.

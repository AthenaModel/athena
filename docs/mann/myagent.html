SYNOPSIS

package require projectlib 6.3.3
namespace import ::projectlib::myagent
  • myagent name ?options...?
  •     -contenttypes list
  • myagent register handler
  • myagent resolve base url
  • object resolve base url
  • object find url
  • object get url ?contentTypes?

DESCRIPTION

In HTTP-parlance, a "user agent" is a program that accesses an HTTP server. An instance of myagent(n) is an object that clients can use to access data from one or more mydomain(i) handlers within the same application. See mydomain(i) for information on what a mydomain(i) is and why it is useful.

In addition, the myagent(n) type is responsible keeping track of the available mydomain(i) instances.

COMMANDS

myagent name ?options...?

Creates a new myagent(n) object called name. The object is represented as a new Tcl command in the caller's scope; myagent returns the fully-qualified form of the name.

Before the new myagent(n) can be used, at least one mydomain(i) must be registered using myagent register.

The myagent(n) may be created with the following options:

-contenttypes list

Specifies the list of content types acceptable to this client, e.g., text/html. Wildcards may be used, e.g., text/*. This list will be passed to the mydomain(i) along with the requested URL; the server will only respond with an acceptable content type.

The mydomain(i) must document the content types it is capable of returning. Usually it will do so dynamically as text/html at the address /urlhelp.

myagent register handler
Registers a mydomain(i) object so that it can be accessed by instances of myagent(n). The handler is the actual object. By convention, an application's main domains are /app and /help.
myagent resolve base url

Given a base URL and some other url, possibly relative, this command resolves the two into a single url. For example, given /app/group and BLUE, this command will return /app/group/BLUE. If the url were /app/nbhood/N1, the returned URL would be /app/nbhood/N1 instead.

The client should always use this command (or, equivalently, the instance's resolve subcommand) on all links embedded in returned content, where base is the URL of the returned content as a whole.

INSTANCE COMMAND

Each instance of myagent(n) has the following subcommands:

object resolve base url
This is identical to myagent resolve; it is provided for convenience.
object find url
Validates the url, verifying its syntax and that its domain is known to the agent. Returns a list, {domain newurl}, where the domain is the domain extracted from the url, e.g., "/app", and the newurl is the url in canonical form.
object get url ?contentTypes?

Retrieves the url's content. If the URL cannot be retrieved, the command will throw an error with error code NOTFOUND; the error message will indicate the reason for the error.

If present, the url's scheme must be "my:".

If one or more contentTypes are given, the returned content will have the first available content type in the list; if none of the the requested content types are available, the server will throw NOTFOUND. If contentTypes is not given, it defaults to the value of the -contenttypes option.

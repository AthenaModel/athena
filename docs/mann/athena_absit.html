DESCRIPTION

This man page describes a component of the athena(n) package that is used to manage and access abstract situation entities.

The component is accessible as a subcommand of the athena(n) and athenadb(n) objects; Public Subcommands are accessible from athena(n) and (usually) from athenadb(n), and Private Subcommands only from athenadb(n).

COMMANDS

Public Subcommands

The adb absit command takes the following subcommands, where adb is an instance of athena(n).

adb absit exists s
Returns 1 if there is an absit with ID s, and 0 otherwise.
adb absit get s ?parm?
By default, returns a dictionary of information about the absit with ID s (the dictionary is consistent with the absits SQL table). If parm is given, it returns the value of that entry in the dict.
adb absit isinitial s
Returns 1 if absit s is in the initial state, and 0 otherwise.
adb absit islive s
Returns 1 if absit s is "live", i.e., it has not been resolved, and 0 otherwise.
adb absit move s location

Moves absit s to the new location, which must be in the same neighborhood. The location may be a lat/long pair in decimal degrees, or an MGRS string. Throws INVALID if the location is invalid or outside the situation's neighborhood.

NOTE: Moving an absit has no effect on the simulation results, but will affect how the absit is displayed on a map.

adb absit names
Returns a list of the IDs of the existing absits.
adb absit validate s
Validates s as an absit ID, returning it if it is valid and throwing INVALID otherwise.
adb absit view s ?tag?
By default, returns a dictionary of information about the absit with ID s, consistent with the fmt_absits SQL view. The tag allows the command to tailor its output for different output formats (e.g., html); at present it is ignored.

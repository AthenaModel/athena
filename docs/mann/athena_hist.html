DESCRIPTION

This man page describes a component of the athena(n) package that is used to access the list of output history variables available for query after a run of the Athena simulation. Since the history is stored in SQL tables, accessing the history itself is best accomplished by using Athena's built in SQL query capability directly on the saved database.

The component is accessible as a subcommand of the athena(n) and athenadb(n) objects; Public Subcommands are accessible from athena(n) and (usually) from athenadb(n), and Private Subcommands only from athenadb(n).

COMMANDS

Public Subcommands

The adb hist command takes the following subcommands, where adb is an instance of athena(n).

adb hist meta huddle
Returns meta data about all history variables as huddle. Each history variable has an associated description and a list of keys with a key label and a list of valid key values for each key. The description and labels are meant for display. The returned data is suitable for conversion to other formats, such as JSON. Rather than return all possible key values for each history table key, only the key values that are present in the table are returned.
adb hist vars
Returns the entire list of history variables and their keys for access to history data. These can be used when constructing an SQL query for the history. For example, one of the returned variable/keys pair is security {n g} indicating that there is a history table called hist_security that contains the history of security by neighborhood and group. The keys to use, if desired, in an SQL query for that history are n and g.
